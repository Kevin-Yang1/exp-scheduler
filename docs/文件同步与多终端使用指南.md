# 文件同步与多服务器终端 · 使用指南

本指南介绍 exp-scheduler 的两项功能：

- **文件同步**：在你的电脑与服务器之间、以及服务器与服务器之间，用 rsync 同步目录或文件（适合在多台 GPU 服务器间搬运模型权重、数据集）。
- **多服务器终端**：为每台服务器开一个真实的交互式 shell，支持"广播"模式同时在多台机器上敲命令（适合统一创建/维护 conda 环境）。

---

## 1. 工作原理：先理解"主控"

打开哪台机器的网页，哪台就是本次操作的**主控（controller）**。所有 SSH 连接都由主控**主动向外发起**，服务器永远不需要回连主控。

```
你的浏览器 ──SSH 隧道──> 主控 (运行 exp-scheduler)
                            │
                            ├─ ssh -p 2222 ──> 服务器 A
                            └─ ssh -p 2200 ──> 服务器 B
```

这带来两个重要结论：

- **主控没有公网 IP 也完全可用**（比如你的本地 WSL / 笔记本）——因为连接都是它主动发出的，没人需要找到它。
- **主控必须能 SSH 直达你想操作的每一台服务器**。主控连不通的服务器无法使用。

两种典型部署：

| 部署方式 | 适用场景 |
|---|---|
| 主控 = 你的本地电脑（WSL/开发机） | 本地能连所有服务器，最灵活，推荐 |
| 主控 = 某台服务器 | 在服务器上常驻；但要确保这台能连通其他所有目标服务器 |

> **选主控的原则**：放在**能 SSH 直达全部目标服务器**的机器上。服务器之间不需要互通——两台互不通时，主控会自动用 SSH 隧道在中间转发（见 [4.3 路由](#43-路由直连优先桥接兜底)）。

## 2. 部署主控

### 2.1 依赖要求

| 组件 | 要求 | 说明 |
|---|---|---|
| 主控的 `ssh` / `ssh-keygen` / `ssh-agent` | OpenSSH ≥ 7.6 | 必需 |
| 主控与服务器两端的 `rsync` | ≥ 3.1 | 文件同步必需 |
| `sshpass` | 可选 | 仅当你用**密码**认证某个节点时，主控上才需要安装 |

### 2.2 状态目录必须在 Linux 原生文件系统

私钥以 `0600` 权限存储、临时 ssh-agent 要创建 unix socket——这两件事在 **NTFS / WSL 的 `/mnt/c`** 上都会失效，导致桥接传输完全不可用。

- WSL 用户：`state_dir` 不要放在 `/mnt/c`，用默认的 `~/.local/share/exp-scheduler` 即可。
- `exp-scheduler doctor` 会检查并在文件系统是 `9p`/`drvfs`/`v9fs` 时告警。

### 2.3 自检

```bash
exp-scheduler doctor
```

关注这几行：

```
ssh: /usr/bin/ssh
rsync: /usr/bin/rsync
ssh-keygen: /usr/bin/ssh-keygen
ssh-agent: /usr/bin/ssh-agent
sshpass: 未安装（可选，仅密码认证的节点需要）
state_dir 文件系统: ext4
```

任一 `ssh/rsync/ssh-keygen/ssh-agent` 显示 `missing` 都会影响功能；`state_dir 文件系统` 若是 `9p/drvfs/v9fs` 会有告警，请按提示把状态目录移到原生分区。

### 2.4 启动并访问

```bash
exp-scheduler serve         # 默认监听 127.0.0.1:17861
```

服务只监听本机回环。从你的电脑访问服务器实例时，用 SSH 隧道转发：

```bash
ssh -L 17861:127.0.0.1:17861 user@server
# 然后浏览器打开 http://127.0.0.1:17861
```

打开后顶部多了「**文件同步**」和「**多终端**」两个标签页。

## 3. 文件同步：准备节点

进入「文件同步」页。页面从上到下分为：传输任务、节点管理、SSH 密钥库、连通性矩阵。**首次使用按"密钥库 → 节点 → 测试连接"的顺序准备。**

> 本机（主控自己）会作为一个名为 **local** 的伪节点自动存在，无需注册。

### 3.1 SSH 密钥库

集中管理私钥，私钥**只存在主控本机**，不会出现在任何 API 响应或日志里。点「SSH 密钥库」区块的 **新增密钥**：

- **密钥名称**：如 `cluster-ed25519`。
- **来源**二选一：
  - **粘贴私钥**：把私钥内容贴进文本框。会以 `0600` 权限保存到主控数据目录。
  - **引用已有路径**：填主控上已存在的私钥路径，如 `/root/.ssh/id_ed25519`（不复制内容，只引用）。

> **限制**：带 passphrase（口令保护）的私钥不支持。可先用 `ssh-keygen -p -N '' -f <私钥>` 去掉口令，或改用"引用路径"配合你自己的 ssh-agent。

> **前提**：注册节点前，请确保该密钥的公钥已经加入目标服务器的 `~/.ssh/authorized_keys`（即 `ssh-copy-id` 已做过、能免密登录）。

### 3.2 注册节点

点「节点管理」区块的 **注册节点**，填写：

| 字段 | 说明 |
|---|---|
| 节点名称 | 显示名，如 `a100-01` |
| 主机地址 | IP 或域名。**必须是主控能直接解析的真实地址**，不要填只存在于 `~/.ssh/config` 里的 Host 别名 |
| SSH 端口 | 非 22 的端口在这里填，如 `2222` |
| 用户名 | 如 `root` |
| 认证方式 | **SSH 密钥**（推荐）或**密码** |

- 选 **SSH 密钥**：从密钥库下拉里选一个。
- 选 **密码**：填登录密码。页面会红字警示：「密码将以明文存储于本机数据库，建议优先使用 SSH 密钥认证」。密码认证依赖主控安装了 `sshpass`。

### 3.3 测试连接

节点列表里每行有 **测试** 按钮。点击后主控会真的 SSH 过去，一次性探测并显示：

- 是否连通（成功显示 `测试通过（XX ms）`，失败显示红色中文原因）；
- 对端 **rsync 版本**；
- **转发 ✓/✕**：sshd 是否允许 TCP 端口转发（桥接做发起端时必需）；
- **Agent ✓/✕**：sshd 是否允许 agent 转发（远程节点做发起端向第三方认证时必需）。

> 测试失败的常见原因会映射成中文，例如"认证失败"（检查密钥/密码）、"网络不可达或 sshd 未运行"、"主机密钥发生变化"。

## 4. 创建与管理传输任务

### 4.1 新建传输

点「传输任务」区块的 **新建传输**：

- **任务名称**：可选，便于在列表里识别。
- **源节点 / 目标节点**：下拉选择（含"本机(local)"）。
- **源路径 / 目标路径**：要传输的目录或文件路径。
- **源目录语义**（关键，决定 rsync 的尾斜杠行为）：
  - **复制目录本身** → 在目标下生成 `目标/<目录名>/...`
  - **仅复制目录内容** → 把源目录里的内容直接铺到目标目录下

### 4.2 高级选项（rsync 参数）

展开「高级选项」：

- ☐ **压缩传输 (-z)**：对已压缩的权重文件（safetensors/ckpt）是浪费 CPU，文本/代码才值得开。
- ☐ **试运行 (dry-run)**：只演示不实际传输，强烈建议第一次或带 `--delete` 时先勾。
- ☐ **删除目标端多余文件 (--delete)**：危险操作。会删掉目标目录里源端没有的文件，提交时弹二次确认：「--delete 会删除目标目录中源端不存在的文件，可能造成数据永久丢失！建议先勾选 dry-run 试运行确认影响范围」。
- **带宽限制 --bwlimit**：如填 `20M` 限速，留空不限。
- **排除规则 --exclude**：如 `*.pyc`、`__pycache__/`，可加多条。
- **其他白名单参数**：如 `--checksum --update`。只接受白名单内的安全参数（`-z/--compress`、`--exclude=`、`--include=`、`--bwlimit=`、`--checksum`、`--update`、`--ignore-existing`、`--no-perms`、`--chmod=`、`--whole-file/--no-whole-file` 等），其他会被拒绝。

### 4.3 路由：直连优先、桥接兜底

选好两端后，页面会自动解析「传输路由」并展示候选。**默认"自动选择"即可**，它的优先级是：

| 路由 | 徽章 | 含义 |
|---|---|---|
| 本机复制 | `本机内复制` | 两端都是本机 |
| 直连·源端发起 | `A ⇒ B` | 源端能直接连到目标端，由源端跑 rsync 推送 |
| 直连·目标端拉取 | `A ⇒ B` | 目标端能直接连到源端，由目标端拉取 |
| 桥接·源端经主控推送 | `A ⇒(经主控)⇒ B` | 两端互不通，主控用 SSH 隧道在中间转发 |
| 桥接·目标端经主控拉取 | `A ⇒(经主控)⇒ B` | 同上，方向相反 |

- **直连**带宽最优，优先选用。
- **桥接**用于两台服务器互相连不通、但主控能连通双方的情况。数据流经主控（不落盘），所以会显示黄色提示：「数据经主控中转，速率受主控带宽限制」——主控是家用上行/笔记本时尤其明显。
- 如果某条路由需要先探测链路连通性，会显示 **待探测**，可点「探测后确定」；不可行的路由会灰显并给出原因。
- 也可手动锁定某条路由（手动指定后不会自动降级）。

> **桥接的前提**：桥接路由要求**对端节点使用密钥认证**（密码无法安全送达发起端），且发起端服务器的 sshd 开启了 `AllowTcpForwarding yes` 和 `AllowAgentForwarding yes`（测试连接里的"转发 ✓"/"Agent ✓"会告诉你）。两端都是密码认证、或两端互不通且无法桥接时，可改用[手动两段法](#52-两台互不通的服务器之间)。

### 4.4 监控、取消、重试、日志

「传输任务」区块分**进行中**和**历史记录**两部分：

- **进行中**：显示路由徽章、状态（排队中/传输中）、进度条 + 百分比 + 速率 + ETA + 已传字节。建桥阶段显示「正在建立连接...」。点 **取消** 二次确认后停止。
- **历史记录**：表格列出状态、路由、耗时、完成时间。每行可：
  - **重试**：用原参数重新填充新建表单（rsync 默认带 `--partial-dir`，可断点续传）。
  - **日志**：打开日志弹窗，可加载完整日志。
  - **删除**：删除记录并清除日志文件。

> **中断后怎么办**：服务重启会把进行中的任务标为 `中断`，主控休眠/断网会标为 `失败`——都**不会自动重跑**（避免 `--delete` 类操作意外重放），手动点"重试"即可续传。

## 5. 典型场景配方

### 5.1 你的电脑 ↔ 服务器

主控就部署在你电脑上。注册服务器为节点后：

- **上传**：源节点选 `本机(local)`，目标节点选服务器 → 自动走"直连·源端发起"（主控本机直接 rsync 推送）。
- **下载**：源节点选服务器，目标节点选 `本机(local)` → 自动走"直连·目标端拉取"。

### 5.2 两台互不通的服务器之间

A、B 互相 SSH 不通，但主控能连通两者：

- 源 A、目标 B，自动路由会选 **桥接**（`A ⇒(经主控)⇒ B`），数据经主控隧道转发。确保 B（对端）是密钥认证、A（发起端）sshd 允许转发。
- 若桥接也不可用（如两端都是密码认证，或 sshd 禁了转发），用**手动两段法**：先建一个 `A → 本机` 的任务，再建一个 `本机 → B` 的任务，主控作为中转盘。

### 5.3 两台能直连的服务器之间

A 能 SSH 到 B：源 A、目标 B 自动走"直连·源端发起"，由 A 直接 rsync 到 B，不经主控，带宽最优。

## 6. 多服务器终端

进入「多终端」页，为每台服务器开一个**真实交互式 shell**（本机是登录 shell，远程是 `ssh -tt` 登录 shell，conda 初始化都会生效）。

### 6.1 新建与操作

- 顶部下拉选节点 → 点 **新建终端**，页面网格里出现一个终端卡片，可像普通终端一样输入、运行、`Ctrl-C`、响应 `y/n` 提示。
- 卡片头部有状态灯（绿=存活、红=断开）、节点名、广播勾选框、关闭按钮。
- 关闭会二次确认：「会话中正在运行的命令会被终止」。

### 6.2 广播模式：同时在多台机器上敲命令

这是统一管理 conda 环境的核心：

1. 打开顶部 **广播输入** 开关（提示：「广播开启时，键盘输入将同步发送到所有勾选的终端」）。
2. 勾选要参与广播的终端卡片上的"广播"复选框。
3. 在任意一个已勾选的终端里输入——命令会同步发到**所有勾选**的终端。比如同时执行：
   ```
   conda create -n myenv python=3.11 -y
   pip install torch transformers
   ```
   `Ctrl-C` 也会一起广播（同时打断所有节点的安装）。
4. 某台报错需要单独处理时：**关闭广播开关**（或取消那台的勾选），单独在那台终端里敲命令修复，修完再加回广播。

### 6.3 会话生命周期

- 刷新页面或网络抖动断开后，**会话不会关闭**，重新打开页面会自动重连并回放最近的输出，正在跑的安装不受影响。
- 远程连接断开时卡片显示「连接已断开」+ **重连** 按钮。
- 无人查看且空闲超过 **30 分钟**（`terminal_idle_timeout_seconds`）的会话自动回收；同时打开的会话数上限 **16**（`max_interactive_terminals`）。
- **服务重启后会话不保留**。
- **长操作建议套 tmux**：大规模 `conda install`、模型下载等，先在终端里 `tmux` 再执行，避免主控休眠或网络断开把进程一起带走。广播模式下中断会在多台节点同时留下装到一半的环境。

### 6.4 conda 环境对比

页面底部「conda 环境对比」面板可展开，点 **刷新** 拉取所有节点的 conda 环境，渲染成矩阵：

- 行 = 环境名，列 = 节点。
- 某台缺失的环境格子标红（`✗`），存在的标 `✓`。
- 列头显示各节点状态：正常 / 无 conda / 超时 / 错误。

一眼就能看出哪台机器缺哪个环境，然后用广播模式补齐。

## 7. 故障排查

传输失败时，历史记录的错误列会给出中文原因。对照表：

| 现象 | 含义 / 处理 |
|---|---|
| 无法登录发起端：认证失败 | 密钥/密码不对，去节点页重新"测试连接"检查 |
| 无法连接发起端：网络不可达或 sshd 未运行 | 检查 host/端口、目标 sshd 是否在跑 |
| 发起端/对端主机密钥发生变化 | 可能重装系统也可能中间人；确认无误后在节点页重新"测试连接"更新记录 |
| 对端密钥认证失败 | 确认该节点配的密钥已加入对端 `~/.ssh/authorized_keys` |
| 发起端 sshd 拒绝建立端口转发 | 端口被占或 `AllowTcpForwarding no`；联系管理员放开，或改手动两段法 |
| 发起端 sshd 禁用了 agent 转发 | `AllowAgentForwarding no`；放开该配置或调换传输方向 |
| 桥接推送/拉取要求对端为密钥认证 | 给对端节点配 SSH 密钥，或调换方向 |
| 目的端磁盘空间不足 | 清理目标盘 |
| 部分文件未能传输 (code 23) | 权限不足或源文件在传输中被改/删，详情看日志 |

连通性矩阵里的单元格：`● 连通（含延迟）`、`✕ 失败（悬停看原因）`、`? 未探测`、`— 不适用`（目标为密码认证、或自己到自己）。点单元格可单边重测，右上角「全矩阵探测」批量探测。

## 8. 已知限制

- **桥接路由下对端必须用密钥认证**（密码无法安全送达发起端）；带 passphrase 的私钥不支持。
- **桥接期间，发起端服务器的 root 可临时借用对端的登录权限**（SSH agent 转发的固有风险，仅限任务运行期间）；可通过选更可信的一端作发起端来缓解。
- **主控连不通的节点无法使用**。注册时可保存（标记为不可达），但要真正能用，需把主控部署到连通性更好的机器。
- **桥接传输速率受主控上下行带宽上限约束**；大文件优先保证两端直连。
- **传输不会自动重跑**：服务重启标 `中断`、连接断开标 `失败`，都需手动"重试"（默认参数支持断点续传）。
- **节点密码以明文存于本机 `scheduler.db`**，请确保状态目录权限受控。
- **多实例共存**（本地和服务器各跑一个）时节点表/密钥库/历史互相独立；指定唯一主实例负责服务器间同步，**同一目标目录绝不允许两个实例并发写入**（带 `--delete` 尤其危险）。
- **终端会话不跨服务重启保留**。
- WSL 主控休眠（合盖/Modern Standby）会冻结所有连接，中断长传输与终端会话；恢复后若系统时间错乱，执行 `sudo hwclock -s` 或启用 systemd + timesyncd。
